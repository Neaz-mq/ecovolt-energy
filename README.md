# EcoVolt — Clean Energy Landing Page Template

A modern, high-performance HTML landing page template built for clean energy, solar, and sustainability businesses. Built with Vite, SCSS (BEM), and vanilla JavaScript — no frameworks, no animation libraries, no bloat.

---

## 📁 File Structure

This repo is the **development project**. When packaged for distribution, it ships as three folders:

```
EcoVolt-Energy/
├── HTML/              → Ready-to-use compiled files (upload this to your host)
│   ├── index.html
│   └── assets/
│       ├── css/        → Compiled stylesheet
│       ├── js/          → Compiled scripts
│       ├── fonts/       → Inter font files + LICENSE.txt
│       └── images/      → Section images
│
├── SourceFiles/        → Editable source (this repo's contents)
│   └── src/
│       ├── index.html
│       ├── scss/
│       │   ├── abstracts/      → _variables.scss, _mixins.scss
│       │   ├── base/           → _reset.scss, _fonts.scss, _typography.scss
│       │   ├── components/     → _buttons.scss
│       │   ├── layout/         → _navbar.scss, _footer.scss
│       │   └── sections/       → one file per section (see table below)
│       ├── js/modules/         → one JS module per section
│       └── assets/             → fonts, icons, section images
│
└── Documentation/      → Full setup & customization guide
```

In this repo specifically:

```
.
├── src/                  → Source files (SCSS, JS, assets) — see structure above
├── public/               → Static files copied as-is into the build (favicons, manifest, logo)
├── documentation/        → Buyer-facing documentation (index.html)
├── scripts/              → Build helper scripts
│   └── copy-license.js   → Copies the Inter font license into dist/ after build
├── dist/                 → Build output (generated by `npm run build`, gitignored)
├── package.json
├── vite.config.js
└── README.md
```

## 🚀 Quick Start

> ⚠️ **Do not open `dist/index.html` (or `HTML/index.html`) by double-clicking it.**
> Modern browsers block ES module scripts from running over `file://` for security
> reasons. Since this template's entrance animations rely on JavaScript, the page
> will appear blank or unstyled if opened this way.

Use one of these instead:

- **VS Code Live Server** — right-click `index.html` → "Open with Live Server"
- **Any local server**, e.g. `python -m http.server` from inside the `dist`/`HTML` folder
- **Upload to real hosting** — the restriction only applies to local `file://` viewing; it works normally once deployed (see the [live demo](https://ecovolt-energy.netlify.app/))

## 🛠 Development Setup

Requires **Node.js** (any recent LTS version) and npm.

```bash
# 1. Install dependencies
npm install

# 2. Start local dev server with live reload
npm run dev

# 3. Build for production (outputs to /dist, includes font license copy)
npm run build
```

`npm run build` runs `vite build` and then `scripts/copy-license.js`, which copies the Inter font's `LICENSE.txt` into `dist/assets/fonts/inter/` — Vite doesn't bundle it automatically since it isn't referenced by any CSS/JS/HTML.

## 🎨 Customization

### Colors
All colors are defined in one file — `src/scss/abstracts/_variables.scss`:

```scss
$clr-primary:        #b8f53d;   // Accent green — buttons, highlights
$clr-bg:              #050f0a;   // Page background
$clr-white:           #f2f7f0;   // Headings / primary text
$clr-muted:           rgba(178, 210, 190, 0.60);  // Body / secondary text
$clr-border-white:    rgba(255, 255, 255, 0.08);  // Subtle borders
```

Change a value once and it updates everywhere it's used across the template.

### Typography
Uses **Inter** — self-hosted, no Google Fonts dependency.
Font files are located in `src/assets/fonts/inter/`.

To use a different font, update `src/scss/base/_fonts.scss` and replace the `.woff2` files.

### Logo & Favicon
Replace these files in the `/public` folder:

| File | Used for |
|---|---|
| `logo.svg` | Navbar / footer logo |
| `favicon-icon.svg` | Browser tab (SVG, modern browsers) |
| `favicon.ico` | Legacy browser fallback |
| `favicon-96x96.png` | Standard favicon |
| `apple-touch-icon.png` | iOS home screen icon |
| `web-app-manifest-192x192.png` / `web-app-manifest-512x512.png` | PWA manifest icons |
| `site.webmanifest` | PWA manifest config |

> **Tip:** Use [realfavicongenerator.net](https://realfavicongenerator.net) to generate a full favicon set from a single image.

### Navigation Links
Edit `.navbar__links` in `src/index.html` to update menu items.
