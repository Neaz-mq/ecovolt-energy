1. Viewing the Template

Important: Do not open index.html by double-clicking it from your file explorer. Modern browsers block local files (file://) from loading stylesheets, scripts, and fonts for security reasons. The page will appear blank or unstyled if viewed this way. Use one of the methods below instead.

Option A — VS Code Live Server (easiest)

  1. Install the "Live Server" extension in VS Code
  2. Right-click HTML/index.html
  3. Select "Open with Live Server"

Option B — Local server (XAMPP / WAMP / MAMP)

  1. Copy the HTML folder into your htdocs (or www) directory
  2. Start Apache from your control panel
  3. Visit http://localhost/HTML/ in your browser

Option C — Upload to live hosting

Once uploaded to any web host, the template works normally — this restriction only applies to local file viewing.

3. Customizing the Template

Editing text and images

All content lives directly in HTML/index.html (or SourceFiles/src/index.html if editing source). Open it in any text editor and replace the placeholder text. To swap images, replace the files inside assets/images/sections/ with your own — keep the same filenames to avoid editing the HTML, or update the src paths if you rename them.

Editing colors

All colors are defined in one file:

SourceFiles/src/scss/abstracts/_variables.scss

The key tokens are:

$clr-primary:  #b8f53d;              // Accent green — buttons, highlights
$clr-bg:       #050f0a;              // Page background
$clr-white:    #f2f7f0;              // Headings / primary text
$clr-muted:    rgba(178,210,190,.60); // Body / secondary text

Change a value once and it updates everywhere across the template.

Editing typography

The template uses Inter, self-hosted with no Google Fonts dependency. Font files are in assets/fonts/inter/. To use a different font, update src/scss/base/_fonts.scss and replace the .woff2 files.

Replacing the logo and favicon

Replace these files in the HTML/ folder (or public/ in the source):

FileUsed for
logo.svgNavbar and footer logo
favicon-icon.svgBrowser tab (modern browsers)
favicon.icoLegacy browser fallback
favicon-96x96.pngStandard favicon
apple-touch-icon.pngiOS home screen icon
web-app-manifest-192x192.png / 512x512.pngPWA manifest icons
site.webmanifestPWA manifest configuration

Tip: Use realfavicongenerator.net to generate a full favicon set from a single image.

Editing navigation links

Edit the .navbar__links list inside index.html to update menu items and anchor targets.

Contact & newsletter forms

Note: The contact form and newsletter signup included in this template are front-end only — they do not send emails out of the box. To make them functional, connect your own backend (e.g. a PHP mail script, or a third-party form service of your choice).

Working with source files (optional)

If you want to edit the SCSS or JavaScript source rather than the compiled output, you'll need Node.js (any recent LTS version) installed. Then from the SourceFiles/ folder:

npm install
npm run dev      # starts a local dev server with live reload
npm run build    # compiles everything into the HTML/ output folder

7. FAQ & Troubleshooting

The page looks blank or unstyled

You most likely opened index.html directly from your file explorer. See Section 1 — use Live Server, a local server, or upload to live hosting instead.

The contact form doesn't send me an email

This is expected — the form is front-end only by default. See Section 3 for how to connect a backend.

Fonts look different after I changed them

Make sure your new .woff2 files are placed in assets/fonts/ and that the paths in src/scss/base/_fonts.scss match the new filenames exactly.

npm run build fails or "command not found"

Make sure Node.js is installed, and that you ran npm install first from inside the SourceFiles/ folder.

My images aren't showing up

Check that replacement images are placed in assets/images/sections/ with filenames matching the src paths used in index.html — file paths are case-sensitive.
