EcoVolt — Documentation

Version 1.0 · Clean Energy Landing Page Template

Thank you for purchasing EcoVolt. This guide covers how to view, customize, and deploy the template.

1. Viewing the Template

Important: Do not open index.html by double-clicking it from your file explorer. Modern browsers block local files (file://) from loading stylesheets, scripts, and fonts for security reasons. The page will appear blank or unstyled if viewed this way. This applies to virtually all modern web templates, not just this one.

Use one of these methods instead:

Option A — VS Code Live Server (easiest)

  1. Install the "Live Server" extension in VS Code
  2. Right-click dist/index.html
  3. Select "Open with Live Server"

Option B — Local server (XAMPP / WAMP / MAMP)

  1. Copy the dist folder into your htdocs (or www) directory
  2. Start Apache from your control panel
  3. Visit http://localhost/dist/ in your browser

Option C — Upload to live hosting

Once uploaded to any web host, the template works normally — this restriction only applies to local file viewing.

2. File Structure

dist/             → Ready-to-use compiled files (upload this to your host)
  ├─ index.html
  ├─ assets/
  │   ├─ css/      → Compiled stylesheet
  │   ├─ js/       → Compiled scripts
  │   ├─ fonts/    → Inter font files
  │   └─ images/   → Template images
  ├─ favicon files, logo.svg, site.webmanifest

src/              → Editable source files (SCSS + JS modules)
documentation/    → This file

3. Customizing the Template

Editing text and images

All content lives directly in dist/index.html (or src/index.html if editing source). Open it in any text editor and replace the placeholder text, then swap image files inside assets/images/ with your own — keep the same filenames to avoid editing the HTML, or update the src paths if you rename them.

Editing colors and design tokens

If you're working from the source files, all colors, spacing, and typography values are centralized in:

src/scss/abstracts/_variables.scss

Change a value once here and it updates everywhere it's used across the template.

Working with source files (optional)

If you want to edit the SCSS or JavaScript source rather than the compiled dist output, you'll need Node.js installed. Then, from the project root:

npm install
npm run dev      # starts a local dev server with live reload
npm run build    # produces an updated dist/ folder when you're done
